# Security Policy

## Scope

CED-AH is a philosophical framework and documentation repository. It contains no executable production code, no deployed services, and no user-facing applications. The `scripts/` directory contains local development tooling only.

## Reporting Concerns

If you discover a security concern related to this repository (e.g., accidentally committed credentials, a vulnerability in CI workflows, or a concern about the project's infrastructure), please report it to:

**research@arqory.ai**

We will acknowledge receipt within 48 hours and provide an initial assessment within one week.

## Infrastructure

For security concerns related to The Arqory Research Institute's commercial infrastructure, deployment platforms, or hosted services, contact:

**security@arqory.ai**

## What This Policy Does Not Cover

This policy does not cover the philosophical content of the CED-AH framework. Challenges to claims, evidence, or arguments should be submitted through the standard [contribution process](CONTRIBUTING.md) using the [Challenge Evidence template](.github/ISSUE_TEMPLATE/challenge-evidence.md).
